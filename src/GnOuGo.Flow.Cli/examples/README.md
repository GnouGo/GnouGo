# `GnOuGo.Flow.Cli` Examples

This folder contains several ready-to-run YAML workflows for `GnOuGo.Flow.Cli`.

## Browser / MCP Examples

### 1. `mcp-browser-navigation-demo.yaml`
The simplest demo to verify `GnOuGo.Browser.Mcp` integration:
- MCP tool discovery
- 3 successive navigations
- reading content at each step
- closing the browser session

```powershell
cd C:\github\GnOuGo.Agent
dotnet run --project src/GnOuGo.Flow.Cli/GnOuGo.Flow.Cli.csproj -- run src/GnOuGo.Flow.Cli/examples/mcp-browser-navigation-demo.yaml
```

### 2. `mcp-browser-form-demo.yaml`
Form demo driven exclusively with generic browser tools:
- `browser_get_content` with `url` to open the page
- `browser_wait`
- `browser_fill`
- `browser_select`
- `browser_click`
- `browser_click_text`
- `browser_get_content`

```powershell
cd C:\github\GnOuGo.Agent
dotnet run --project src/GnOuGo.Flow.Cli/GnOuGo.Flow.Cli.csproj -- run src/GnOuGo.Flow.Cli/examples/mcp-browser-form-demo.yaml
```

### 3. `browser-homepage-menu-scrape.yaml`
A more "agentic" navigation example:
- opens a homepage via `browser_get_content(url: ...)`
- reads the text and HTML
- lets the LLM extract useful links
- then visits the selected pages
- synthesizes the site

> Requires a valid LLM configuration for `llm.call`.

```powershell
cd C:\github\GnOuGo.Agent
dotnet run --project src/GnOuGo.Flow.Cli/GnOuGo.Flow.Cli.csproj -- run src/GnOuGo.Flow.Cli/examples/browser-homepage-menu-scrape.yaml
```

### 4. `slimfaas-browser-research-agent.yaml`
Main autonomous research example focused on `SlimFaas`:
- research brief generated by the LLM
- multi-source catalogue (official site, docs, blog, Google, Bing, DuckDuckGo, Reddit, LinkedIn, GitHub, Hacker News, News)
- reading both text and rendered HTML to better diagnose potential overlays / banners
- multi-round planning
- structured extraction of found signals
- structured final summary + narrative

> No automatic cookie click: the client/LLM decides, from the rendered HTML, whether to click a button.

> Requires a valid LLM configuration for `llm.call`.

```powershell
cd C:\github\GnOuGo.Agent
dotnet run --project src/GnOuGo.Flow.Cli/GnOuGo.Flow.Cli.csproj -- run src/GnOuGo.Flow.Cli/examples/slimfaas-browser-research-agent.yaml
```

Example with another company:

```powershell
cd C:\github\GnOuGo.Agent
dotnet run --project src/GnOuGo.Flow.Cli/GnOuGo.Flow.Cli.csproj -- run src/GnOuGo.Flow.Cli/examples/slimfaas-browser-research-agent.yaml -i subject="Acme Cloud" -i company_domain="acme.com"
```

## Notes

- `GnOuGo.Browser.Mcp` must be configured in `appsettings.json` on the CLI/Server side.
- Playwright binaries must be installed for `src/GnOuGo.Browser.Mcp`.
- Workflows using `llm.call` require a valid LLM configuration.
- Cookie click logic or link extraction is no longer implemented on the MCP server side: it is now handled by the client/LLM when needed.
